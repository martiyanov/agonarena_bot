## 🐛 Баги AG-017

### Баг #1: Дублирование "Раунд 1: Раунд 1:"

**ID:** AG-017-BUG-1  
**Серьёзность:** Средняя (UX)  
**Статус:** ✅ Исправлено

**Описание:**
В выводе судей префикс "Раунд 1:" дублировался.

**Было:**
```
Раунд 1: Раунд 1: Стороны установили контакт...
```

**Стало:**
```
Раунд 1: Стороны установили контакт...
```

**Причина:**
- `judge_service.py` fallback уже содержит префикс "Раунд 1:" в `round1_comment`
- `_format_final_verdict()` добавлял префикс ещё раз

**Root Cause:** Недостаточная проверка формата данных между сервисами

**Fix:**
- Удалено дублирование в `_format_final_verdict()` (строки 106-108)
- Удалено дублирование в `_show_results()` (строки 593-595)

**Prevention Measure:**
- [ ] Добавить unit тест на форматирование судей
- [ ] Документировать формат `round1_comment` в интерфейсе JudgeService

**Урок:** Тестировать вывод судей на реальных данных перед релизом

### Баг #2: Кнопка "Сценарии" → приветствие вместо пикера

**ID:** AG-017-BUG-2  
**Серьёзность:** Высокая (ломает основной flow)  
**Статус:** ✅ Исправлено

**Описание:**
При нажатии кнопки "📚 Сценарии" показывалось приветствие вместо пикера сценариев.

**Причина:**
Конфликт handlers! Кнопка была зарегистрирована в **двух** обработчиках:
1. `@router.message(F.text == SCENARIOS_BUTTON)` → `_send_scenario_picker()` ✅
2. `@router.message(F.text.in_({RULES_BUTTON, RULES_BUTTON_LEGACY, SCENARIOS_BUTTON}))` → `_show_rules()` ❌

Второй handler перехватывал кнопку.

**Root Cause:** Недостаточный аудит handlers при добавлении новых кнопок

**Fix:**
- Удалено `SCENARIOS_BUTTON` из второго handler (строка 1149)

**Prevention Measure:**
- [ ] Вести реестр всех handlers и кнопок
- [ ] Добавить тест на отсутствие конфликтов handlers
- [ ] Code review на наличие конфликтов (одна кнопка — один handler)

**Урок:** Проверять все handlers на конфликты перед релизом

### Баг #3: Нет inline кнопки "Завершить раунд" в сообщении "Поединок уже идёт"

**ID:** AG-017-BUG-3  
**Серьёзность:** Высокая (ломает основной flow)  
**Статус:** 🔄 В очереди на исправление

**Описание:**
Сообщение "Поединок уже идёт" содержит текст про inline-кнопку, но самой кнопки нет.

**Скриншот:** Пользователь прислал — текст есть, кнопки нет

**Причина:**
В `show_main_menu()` добавлен текст про inline-кнопку, но `reply_markup=build_in_duel_keyboard()` не вызывается.

**Root Cause:** Неполная реализация — добавлен текст, но не добавлена сама кнопка

**Fix (планируется):**
- Добавить `build_in_duel_keyboard()` к сообщению
- Получить `duel_id` и `round_number` из `DuelService`

**Prevention Measure:**
- [ ] Не добавлять текст про кнопку без самой кнопки
- [ ] Тестировать UI полностью, не только текст

### Баг #6: Кнопка от старой дуэли остаётся активной (КРИТИЧНО)

**ID:** AG-017-BUG-6  
**Серьёзность:** P0 (критичный)  
**Статус:** 🔄 В очереди на исправление

**Описание:**
При начале новой дуэли inline кнопка "🏁 Завершить раунд" от старой дуэли остаётся активной. При нажатии — ошибка "Эта кнопка от старой дуэли".

**Пользователь не должен видеть эту ошибку никогда!**

**Причина:**
Старые inline кнопки не инвалидируются при начале новой дуэли.

**Решения:**
1. При начале новой дуэли — редактировать/удалять старые сообщения с кнопками
2. В callback мягко отклонять старые кнопки (без ошибки пользователю)
3. Оба варианта вместе
