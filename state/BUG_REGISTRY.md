## Процесс добавления багов

1. **ANALYST/ARCH находит проблему** → создаёт запись в этом файле
2. **PM оценивает RICE** → приоритизирует (P0/P1/P2/P3)
3. **DEV берёт из очереди** → фиксит по порядку приоритета
4. **TEST валидирует** → закрывает баг
