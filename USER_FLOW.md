# Agon Arena — USER_FLOW

## Основной пользовательский сценарий MVP

1. Пользователь открывает Telegram-бота и нажимает `/start`.
2. Бот показывает главное меню.
3. Пользователь нажимает `🎯 Выбрать сценарий`.
4. Бот отправляет сообщение:
   `Выберите готовый сценарий для поединка`
5. Бот показывает 10 готовых сценариев в чате.
6. У каждого сценария есть inline-кнопка `Начать сценарий`.
7. Пользователь нажимает `Начать сценарий` у нужного сценария.
8. Бот запускает обычный duel-flow.
9. Раунд 1: пользователь делает ходы текстом или голосом.
10. Пользователь нажимает `🏁 Завершить раунд`.
11. Бот переводит поединок во 2-й раунд со сменой ролей.
12. Раунд 2: пользователь делает ходы.
13. Пользователь снова нажимает `🏁 Завершить раунд`.
14. Бот завершает поединок и показывает итоговый разбор судей.

## Дополнительные сценарии MVP

### Случайный сценарий
- Пользователь нажимает `🎲 Случайный сценарий`
- Бот стартует обычный duel-flow по случайному активному сценарию

### Свой сценарий
- Пользователь нажимает `🎭 Свой сценарий`
- Описывает ситуацию текстом или голосом
- Бот превращает описание в сценарий и запускает обычный duel-flow

## Ожидаемое поведение UX
- Пользователь не вводит код сценария вручную в обычном happy-path
- Выбор сценария делается явно из списка
- Внутри поединка нет двух разных action-кнопок на переход/финиш
- Повторное нажатие `Завершить раунд` не должно ломать состояние
