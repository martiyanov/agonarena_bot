## ЦЕЛЬ

Документ описывает реализацию автоматического спавна TEST субагентов при получении маркера `READY_FOR_TEST: yes` от DEV субагентов, а также fallback таймер в 5 секунд.

---

## ТЕКУЩЕЕ СОСТОЯНИЕ

### Проблема
- Pipeline застревает после завершения DEV этапа
- TEST субагент не спавнится автоматически
- Требуется ручной запуск TEST, что нарушает автоматический workflow

### Ожидаемое поведение
```
PM → DEV → [READY_FOR_TEST: yes] → [auto spawn TEST] → TEST → OWNER_SUMMARY
```

---

## РЕАЛИЗАЦИЯ

### 1. Мониторинг сессий
- Скрипт `scripts/auto_test_spawner.py` мониторит сессии на наличие маркера `READY_FOR_TEST: yes`
- Проверяет выход DEV субагентов в сессионных файлах
- Использует регулярное выражение для поиска маркера: `r'READY_FOR_TEST:\s*yes'`

### 2. Авто-спавн TEST
- При обнаружении маркера, автоматически вызывает спавн TEST субагента
- Использует модель `modelstudio/qwen3-coder-plus` как указано в политике
- Интегрируется с существующим механизмом `sessions_spawn`

### 3. Fallback таймер
- Запускает таймер на 5 секунд при обнаружении DEV активности
- Если TEST не спавнится в течение 5 секунд, принудительно запускает спавн
- Предотвращает застревание pipeline при сбоях в детекции маркера

### 4. Интеграция
- Скрипт может быть запущен как фоновый процесс
- Периодически сканирует сессионные файлы на предмет DEV завершения
- Совместим с существующей архитектурой OpenClaw

## ТЕСТИРОВАНИЕ

### Сценарии:
1. **Нормальный случай**: DEV завершает работу с `READY_FOR_TEST: yes` → TEST спавнится автоматически
2. **Fallback**: DEV завершает работу, но TEST не спавнится вручную → через 5 секунд TEST спавнится автоматически
3. **Ошибка детекции**: Маркер не обнаружен, но есть DEV активность → fallback срабатывает через 5 секунд

### Проверка:
- Логи спавнера показывают обнаружение маркера
- TEST субагент действительно создается после DET
- Время между DEV завершением и TEST стартом ≤ 5 секунд

```
DEV Output:
```

→ Автоматический спавн TEST субагента

### Fallback кейс:

```
DEV Output:
- Made changes to files  
- Updated state/TODO.md
- [no READY_FOR_TEST marker detected due to parsing issue]
```
→ Через 5 секунд принудительный спавн TEST субагента

## ПОЛИТИКА

### DEV обязанности:
- DEV субагент должен завершать свой ответ маркером: `READY_FOR_TEST: yes`
- Маркер должен быть четко виден в выходных данных
- Маркер может быть в любом регистре: `READY_FOR_TEST: yes`, `ready_for_test: YES`, etc.

### Обработка:
- Один TEST субагент спавнится на один успешный DEV сессионный файл
- Избегается дублирование спавнов для одной и той же сессии
- Сессии обрабатываются только один раз
