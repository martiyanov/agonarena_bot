# UX-анализ: Инлайн-кнопка "Завершить раунд"

**Дата:** 2026-03-29  
**Аналитик:** ChatGPT-5.4  
**Статус:** ✅ Готово к реализации

---

## 📋 Решение

**Цель:** Сделать inline-кнопку завершения основным способом выхода из раунда.

**Изменение:** Убрать 🏁 Завершить раунд из ReplyKeyboardMarkup и перенести action в InlineKeyboardMarkup у сообщений дуэли.

---

## 🎯 Ключевые решения

### 1. Где показывать кнопку

| Тип сообщения | Кнопка |
|---------------|--------|
| Старт раунда 1 (после opening line) | 🏁 Завершить раунд |
| После каждого ответа AI в раунде 1 | 🏁 Завершить раунд |
| Переход к раунду 2 (смена ролей) | 🏁 Завершить поединок |
| После каждого ответа AI в раунде 2 | 🏁 Завершить поединок |

### 2. Вариант реализации

**MVP:** Вариант A — кнопка на каждом релевантном сообщении
- Проще внедрить
- Кнопка почти всегда на экране
- Нужна жёсткая state-проверка

**Позже:** Вариант B — только одна актуальная кнопка
- Чище UX
- Меньше дублирования
- Требует хранения message_id последнего actionable сообщения

### 3. CallbackData структура

```
duel:end:{duel_id}:{round_no}
```

Примеры:
- `duel:end:128:1` — завершение раунда 1
- `duel:end:128:2` — завершение раунда 2 (поединка)

**Зачем:** Чтобы различать актуальный ли это поединок и раунд.

### 4. Тексты кнопок

| Раунд | Текст кнопки |
|-------|--------------|
| 1 | 🏁 Завершить раунд |
| 2 | 🏁 Завершить поединок |

### 5. Что убрать из ReplyKeyboard

**Было:**
- 🏁 Завершить раунд
- 🎯 Сценарии | 🎲 Случайный
- ℹ️ Справка | 💬 Отзыв

**Стало (во время дуэли):**
- 🎯 Сценарии | 🎲 Случайный
- ℹ️ Справка | 💬 Отзыв

---

## ⚠️ Edge Cases и поведение

| Ситуация | Ответ пользователю |
|----------|-------------------|
| Двойное нажатие | "Раунд уже завершён" |
| Нажатие по старой кнопке | "Эта кнопка уже неактуальна" |
| Нет активного раунда | "Сейчас нет активного раунда" |
| После истечения времени | "Время уже истекло. Подвожу итог." |
| Во время обработки голосового | "Сначала дождитесь обработки текущего сообщения" |

---

## 🔧 Server-side логика

### Проверки в обработчике callback (по порядку)

1. Есть ли активная дуэль у пользователя
2. Совпадает ли duel_id
3. Совпадает ли текущий раунд
4. Не в состоянии ли дуэль уже finished/transitioning
5. Не идёт ли voice/STT/AI-processing
6. Выполнить transition

### Состояния дуэли

```
idle
round_1_active
round_1_processing
round_1_transition
round_2_active
round_2_processing
round_2_transition
finished
```

**Кнопка end допустима только в:** `round_1_active`, `round_2_active`
