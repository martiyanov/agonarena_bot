# Задача UX-аналитику: Инлайн-кнопка "Завершить раунд"

**Дата:** 2026-03-29  
**Приоритет:** Высокий  
**Статус:** 🟡 Ожидает UX-ревью

---

## 📋 Контекст

Бот для тренировки управленческих поединков (Agon Arena):
- 2 раунда с обменом ролями
- После каждого раунда — разбор от 3 судей
- Пользователь общается с AI-соперником текстом или голосом

---

## 🐛 Проблема

Сейчас кнопка **"🏁 Завершить раунд"** находится в **нижнем меню** (ReplyKeyboardMarkup).

**Что не так:**
1. Нижние кнопки скрываются, когда пользователь начинает переписываться с ботом
2. В Telegram при активной переписке видны только **инлайн-кнопки** (в сообщении)
3. Кнопка не видна в самый важный момент — когда нужно завершить раунд
4. Пользователь может не найти кнопку и зависнуть в процессе

**Скриншоты:** См. в чате от 2026-03-29 06:17 (3 файла)

---

## 🎯 Предложенное решение

Перенести кнопку "Завершить раунд" из нижнего меню в **инлайн-кнопки** к каждому сообщению раунда.

### Где добавить инлайн-кнопку:

1. **Стартовое сообщение дуэли** (после opening line соперника)
   - Функция: `_start_duel()` в `app/bot/handlers/menu.py`

2. **Каждое сообщение с ответом соперника**
   - Функция: `_run_turn()` в `app/bot/handlers/menu.py`
   - После реплики AI показывать кнопку завершения

3. **Сообщение перехода ко второму раунду**
   - Функция: `_process_end_round()` в `app/bot/handlers/menu.py`

### Что убрать:

- Кнопку "🏁 Завершить раунд" из **нижнего меню** (`build_main_menu()` в `app/bot/keyboards/main_menu.py`)
- Это освободит место для других кнопок

## 📐 Технические детали

### Текущая структура меню (`main_menu.py`):

```python
def build_main_menu(has_active_duel: bool = False) -> ReplyKeyboardMarkup:
    if has_active_duel:
        # 5 кнопок в нижнем меню
        keyboard = [
            [KeyboardButton(text="🏁 Завершить раунд")],  # ← УБРАТЬ
            [KeyboardButton(text="🎯 Сценарии"), KeyboardButton(text="🎲 Случайный")],
            [KeyboardButton(text="ℹ️ Справка"), KeyboardButton(text="💬 Отзыв")],
        ]
    else:
        # 4 кнопки в нижнем меню
        keyboard = [
            [KeyboardButton(text="🎯 Сценарии"), KeyboardButton(text="🎲 Случайный")],
            [KeyboardButton(text="ℹ️ Справка"), KeyboardButton(text="💬 Отзыв")],
        ]
```

### Предлагаемая инлайн-кнопка:

```python
from aiogram.types import InlineKeyboardButton, InlineKeyboardMarkup

def build_in_duel_keyboard(duel_id: int, round_number: int) -> InlineKeyboardMarkup:
    """Inline keyboard with end round button for active duel messages."""
    keyboard = [
        [InlineKeyboardButton(
            text="🏁 Завершить раунд", 
            callback_data=f"end_round:{duel_id}:{round_number}"
        )],
    ]
    return InlineKeyboardMarkup(inline_keyboard=keyboard)
```

## ⚠️ Вопросы для UX-аналитика

### 1. Не сломает ли это механику?

- Сейчас кнопка одна на всех (в нижнем меню)
- Инлайн-кнопка будет привязана к конкретному сообщению
- Нужно проверить: callback handler `_process_end_round()` уже обрабатывает `callback_data="end_round"`
- Возможно, потребуется обновить логику обработки callback

### 2. Дублирование кнопок?

- Оставить ли кнопку в нижнем меню **и** добавить инлайн?
- Или **только** инлайн (рекомендуется для чистоты UX)?
- Аргумент за инлайн: видна всегда, не скрывается
- Аргумент за нижнее: привычнее для некоторых пользователей

### 3. Текст кнопки

- Сейчас: "🏁 Завершить раунд"
- Альтернативы: "🏁 Конец раунда", "⏹️ Завершить", "➡️ Дальше"
- Нужно ли менять текст для второго раунда? ("🏁 Завершить поединок")

### 4. Визуальное оформление

- Одна кнопка под сообщением — ок?
- Или добавить другие инлайн-кнопки (например, "Меню", "Справка")?
- Не будет ли слишком много кнопок под каждым сообщением?

### 5. Состояния и edge cases

- Что если пользователь нажмёт кнопку дважды быстро?
- Что если нажмёт после истечения времени раунда?
- Что если нажмёт, когда раунд ещё не начался (pending)?
